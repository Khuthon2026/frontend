# 성과 문서화 가이드 (PT용)

> 팀장이 "문서화해줘"라고 요청할 때만 실행합니다.
> 그 시점까지 구현된 내용을 코드베이스 기반으로 한 번에 정리하세요.
> PT에서 그대로 읽을 수 있도록 한국어로 작성합니다.

## 문서 구조

docs/
├── ux-metrics.md          # 화면 수, 사용자 플로우, 인터랙션

└── feature-summary.md     # 구현 기능 목록 및 기술적 의사결정

## 작성 원칙
- 숫자로 말하기: "여러 페이지" ❌ → "총 5개 페이지, 12개 컴포넌트" ✅
- Before/After가 있으면 반드시 비교 형태로 작성
